# GreenPlanetMart Relational Database Project Brief

## Task Overview

As part of an actual project, you are significantly involved in the planning, development, and implementation of a relational database solution aimed at transforming GreenPlanetMart’s data management processes. This is a project for university and is not supposed to be a wholistic approach but rather a PoC with the data on hand.

Your development and implementation task involves the following steps.

## Development and Implementation Steps

### 1. Requirement Analysis

Conduct a thorough requirement analysis to identify the specific data processing needs of GreenPlanetMart.

Consider requirements from:

- Supply chain management
- Business analytics
- Operational efficiency

### 2. Data Modeling

Develop data models that adequately represent the structure and relationships of GreenPlanetMart’s data.

Consider:

- Capturing and integrating data from various sources
- Requirements for data analysis and reporting

### 3. Technology Selection

Choose an appropriate database management platform for implementing the relational database solution.

Consider factors such as:

- Performance
- Scalability
- Reliability
- Cost

### 4. Data Integration

Integrate data from various sources into the relational database, including:

- Suppliers
- Warehouses
- Distribution centers
- Retail stores

Ensure that the data is:

- Correctly structured
- Cleansed
- Indexed to enable efficient querying and analysis

### 5. Data Reporting

Model and implement an appropriate Business Intelligence data model.

Transform the integrated data into this denormalized structure and connect to this structure using a Business Intelligence tool.

Create 3–5 reports that support GreenPlanetMart.

## Project Report Requirements

Your project report should detail the following aspects.

### 1. Technical Architecture

Thoroughly illuminate the technical architecture of the relational database solution.

Discuss:

- The choice of database management system (DBMS)
- Database design considerations
- How data modeling is conducted to meet the company’s requirements

### 2. Data Integration

Describe how various data sources are integrated into the relational database, including supply chain data from:

- Suppliers
- Warehouses
- Distribution centers
- Retail stores

Discuss measures for:

- Data structuring
- Normalization
- Indexing to optimize query performance

### 3. Challenges and Strategies

Discuss the challenges encountered during the implementation of the relational database solution and the strategies developed to overcome them.

Address technical challenges related to:

- Database design
- Performance optimization
- Scalability

Address organizational challenges related to:

- Change management
- Stakeholder engagement

### 4. Project Management

Detail the project management and execution methods applied to steer and monitor the implementation project.

Provide insights into:

- Project phases
- Timeline
- Resource planning
- Stakeholder management

The goal is to ensure successful delivery.

### 5. Outcomes and Evaluation

Evaluate the outcomes of the implementation project.

Discuss how the relational database solution has improved:

- Data management
- Decision-making processes at GreenPlanetMart

Assess:

- Return on investment (ROI)
- Other benefits achieved through the implementation

### 6. Implementation

Describe the individual steps of implementation and justify your decisions.

Prove the implementation with screenshots of:

- Data models
- Sample data sets
- Meta-information of the database
- Reports
- ETL processes
- Other relevant implementation evidence
